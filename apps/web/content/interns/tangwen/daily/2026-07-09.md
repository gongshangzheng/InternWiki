## 今日完成

在 [[project:digital-human]] 项目上，今天的主线是**将各种模型适配接入 Speed Run**，同时花了很多时间在环境配置上。

### Speed Run 模型适配

1. **Speed Run 线路建设**：在评测框架中增加了快速运行模式，遍历 `evaluate_simple9` 数据集跑通模型推理并输出对比视频，用于主观效果检验。目前已编写 12 个模型的 speed run 适配脚本（artalk_gagavatar / artalk_uika / avatarforcing / ditto / flash_head / flexavatar / joyvasa / lhm / liveportrait / musetalk / unils_gagavatar / wave2lip）。
2. **Speed Run WebUI**：在 WebUI 中添加 Speed Run Tab，支持模型切换、素材选择器、样本 Tab 化展示、参考帧策略选择。
3. **跑通评测代码全流程**：[[task:digital-human/t3-6]] 端到端跑通评测代码的完整流程，验证各环节可用。
4. **评测功能封装为 agent skill**：将评测框架部分功能封装为 skill，agents 可快速调用。
5. **模型参数重新下载**：之前下载的模型参数被误覆盖，已重新下载完成。

### 环境配置（今天最大的耗时点）

6. **UV 分层环境管理机制**：[[task:digital-human/t3-10]] 3DGS 等模型在现有环境中跑不通，花了大量时间排查依赖冲突。为此构建了基于 UV 的基准环境 + 微调环境双层策略：基准环境（`.venv`）覆盖大多数模型；依赖冲突的模型创建增量微调环境（`.envs/{model}`）。包含 env-manager skill、manage_env.sh 脚本、四步决策流程（基准测试→调整基准→git 回退→创建微调）、模型依赖冲突分析。但**目前很多模型的环境还没有完全配好**，特别是 3DGS 相关模型。

### FlexAvatar 接入 + 3DGS 调研

7. **FlexAvatar 适配器接入**：[[task:digital-human/t-1651]] 创建 FlexAvatar（CVPR 2026）模型适配器，支持 pixel3dmm 和 artalk 两种驱动模式，实现七阶段生命周期接口。
8. **3DGS 方法调研**：[[task:digital-human/t-1652]] 调研 FlexAvatar / HyperGaussians / MATCH / AniGS 四个 3DGS 头部 Avatar 方法。结论：FlexAvatar 已接入；HyperGaussians 可作为增强模块；MATCH 需多视角输入且依赖 SM90 架构，不接入；AniGS 为全身 Avatar 方向，作技术参考。
